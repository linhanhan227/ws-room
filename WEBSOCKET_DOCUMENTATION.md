# WebSocket 聊天室接口文档

## 概述

WebSocket 接口提供实时双向通信能力，支持用户加入房间、发送消息、管理用户等功能。

## 连接信息

### WebSocket URL
```
ws://localhost:8080/ws/chat
```

### 连接参数

#### Token 认证

- **token**: JWT Token（通过URL参数传递）
  - 类型：String
  - 必填：是
  - 说明：通过登录接口获取的JWT Token，用于身份认证
  - 格式：Bearer Token，通过URL参数传递
  - 示例：`ws://localhost:8080/ws/chat?token=eyJhbGciOiJIUzI1NiJ9...`
  - 注意：Token默认有效期为7天，过期后需要重新登录

## 消息类型

### 1. JOIN - 加入房间

#### 功能说明

将当前用户加入指定的聊天房间。加入房间后会收到最近10条历史消息，房间内其他用户会收到系统通知。

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 约束条件 | 默认值 |
|--------|------|------|------|---------|--------|
| type | String | 是 | 消息类型，固定值 | 固定为 "JOIN" | - |
| roomId | String | 是 | 房间ID | 纯数字字符串，长度1-20 | - |
| content | String | 否 | 房间密码 | 私有房间必填，公开房间可选 | - |

#### 参数详细说明

**type**
- 固定值："JOIN"
- 说明：标识此消息为加入房间请求
- 必填：是

**roomId**
- 类型：String
- 说明：要加入的房间ID
- 必填：是
- 格式：纯数字字符串
- 长度：1-20个字符
- 约束：房间必须存在且处于活跃状态
- 获取方式：通过 ROOM_LIST 消息获取房间列表

**content**
- 类型：String
- 说明：房间密码
- 必填：否（私有房间必填）
- 约束：
  - 私有房间（isPrivate=true）：必须提供正确的密码
  - 公开房间（isPrivate=false）：无需提供密码
- 长度：1-100个字符
- 注意：密码错误时会返回错误消息

#### 响应参数

| 参数名 | 类型 | 说明 | 可能的值 |
|--------|------|------|---------|
| type | String | 消息类型 | "JOIN" |
| content | String | 加入结果描述 | "成功加入房间: {房间名称}" |
| roomId | String | 房间ID | 与请求参数一致 |
| timestamp | Long | 时间戳（毫秒） | Unix时间戳 |

#### 响应详细说明

**type**
- 固定值："JOIN"
- 说明：标识此响应为加入房间成功

**content**
- 类型：String
- 说明：加入房间的结果描述
- 格式："成功加入房间: {房间名称}"
- 示例："成功加入房间: 测试聊天室"

**roomId**
- 类型：String
- 说明：成功加入的房间ID
- 与请求参数中的roomId一致

**timestamp**
- 类型：Long
- 说明：服务器处理请求的时间戳
- 单位：毫秒
- 格式：Unix时间戳

#### 系统广播

加入房间成功后，房间内其他用户会收到系统消息：

| 参数名 | 类型 | 说明 |
|--------|------|------|
| type | String | "SYSTEM" |
| content | String | "{username} 加入了房间" |
| roomId | String | 房间ID |
| timestamp | Long | 时间戳 |

#### 历史消息

加入房间后会收到最近10条历史消息，每条消息包含：

| 参数名 | 类型 | 说明 |
|--------|------|------|
| type | String | "CHAT" |
| content | String | 消息内容 |
| sender | String | 发送者用户名 |
| roomId | String | 房间ID |
| messageId | String | 消息ID |
| timestamp | Long | 时间戳 |
| isRecalled | Boolean | 是否已撤回 |

#### 错误响应

| 错误信息 | 说明 | HTTP状态码 |
|---------|------|-----------|
| 请指定房间ID | 未提供roomId参数 | - |
| 房间不存在 | 指定的房间不存在或已被删除 | - |
| 房间密码错误 | 私有房间密码错误 | - |
| 房间已满 | 房间用户数已达上限 | - |

### 3. CHAT - 发送聊天消息

#### 功能说明

向当前房间发送聊天消息。消息会经过敏感词检测，包含敏感词的消息会被拒绝。消息会广播给房间内所有用户（包括发送者）。

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 约束条件 | 默认值 |
|--------|------|------|------|---------|--------|
| type | String | 是 | 消息类型，固定值 | 固定为 "CHAT" | - |
| content | String | 是 | 消息内容 | 不能为空，不能包含敏感词 | - |

#### 参数详细说明

**type**
- 固定值："CHAT"
- 说明：标识此消息为聊天消息
- 必填：是

**content**
- 类型：String
- 说明：要发送的聊天消息内容
- 必填：是
- 约束：
  - 不能为空
  - 不能包含敏感词（会被系统拒绝）
  - 长度：1-1000个字符
  - 支持中文、英文、数字、常用标点符号
- 注意：包含敏感词的消息会被拒绝，并返回错误信息

#### 响应参数

| 参数名 | 类型 | 说明 | 可能的值 |
|--------|------|------|---------|
| type | String | 消息类型 | "CHAT" |
| content | String | 消息内容 | 与请求参数一致 |
| sender | String | 发送者用户名 | 当前用户的用户名 |
| roomId | String | 房间ID | 当前所在房间的ID |
| messageId | String | 消息ID | 纯数字字符串 |
| timestamp | Long | 时间戳（毫秒） | Unix时间戳 |
| isRecalled | Boolean | 是否已撤回 | false |

#### 响应详细说明

**messageId**
- 类型：String
- 说明：消息的唯一标识符
- 格式：纯数字字符串
- 用途：用于撤回消息时指定要撤回的消息
- 注意：每个消息都有唯一的messageId

**isRecalled**
- 类型：Boolean
- 说明：消息是否已被撤回
- 可能值：true（已撤回）、false（未撤回）
- 默认值：false

#### 系统广播

消息会广播给房间内所有用户（包括发送者），所有用户收到的消息内容相同。

#### 注意事项

1. **敏感词检测**：所有聊天消息都会经过敏感词检测，包含敏感词的消息会被拒绝
2. **广播机制**：消息会广播给房间内所有用户（包括发送者）
3. **消息撤回**：发送者可以在2分钟内撤回消息
4. **禁言限制**：被禁言的用户无法发送聊天消息
5. **房间限制**：必须先加入房间才能发送聊天消息

#### 错误响应

| 错误信息 | 说明 | 解决方法 |
|---------|------|---------|
| 请先加入房间 | 当前用户未加入任何房间 | 先发送 JOIN 消息加入房间 |
| 您已被禁言 | 用户被禁言，无法发送消息 | 等待禁言结束或联系管理员 |
| 消息包含敏感词 | 消息包含敏感词 | 移除敏感词后重新发送 |
| 消息内容不能为空 | content参数为空 | 提供有效的消息内容 |

### 4. CREATE_ROOM - 创建房间（管理员）

#### 功能说明

创建新的聊天房间。仅管理员可用。创建成功后会自动发送 ROOM_LIST 消息更新房间列表。

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 约束条件 | 默认值 |
|--------|------|------|------|---------|--------|
| type | String | 是 | 消息类型，固定值 | 固定为 "CREATE_ROOM" | - |
| content | String | 是 | 房间名称 | 不能为空，长度1-50字符 | - |
| description | String | 否 | 房间描述 | 长度0-200字符 | - |
| maxUsers | Integer | 否 | 最大用户数 | 1-1000之间的整数 | 50 |
| isPrivate | Boolean | 否 | 是否为私有房间 | true或false | false |
| password | String | 否 | 房间密码 | 私有房间必填，长度1-50字符 | - |

#### 参数详细说明

**type**
- 固定值："CREATE_ROOM"
- 说明：标识此消息为创建房间请求
- 必填：是

**content**
- 类型：String
- 说明：房间名称
- 必填：是
- 约束：
  - 不能为空
  - 长度：1-50个字符
  - 不能包含特殊字符
  - 房间名称在系统中必须唯一
- 注意：房间名称已存在时会返回错误

**description**
- 类型：String
- 说明：房间描述
- 必填：否
- 约束：
  - 长度：0-200个字符
  - 支持中文、英文、数字、常用标点符号
- 默认值：空字符串

**maxUsers**
- 类型：Integer
- 说明：房间最大用户数
- 必填：否
- 约束：
  - 范围：1-1000
  - 必须为整数
- 默认值：50
- 注意：当房间用户数达到上限时，新用户无法加入

**isPrivate**
- 类型：Boolean
- 说明：是否为私有房间
- 必填：否
- 可能值：true（私有）、false（公开）
- 默认值：false
- 约束：
  - true：需要密码才能加入
  - false：任何人都可以加入

**password**
- 类型：String
- 说明：房间密码
- 必填：否（私有房间必填）
- 约束：
  - 私有房间（isPrivate=true）：必须提供密码
  - 公开房间（isPrivate=false）：无需提供密码
  - 长度：1-50个字符
- 注意：密码错误时用户无法加入房间

#### 响应参数

| 参数名 | 类型 | 说明 | 可能的值 |
|--------|------|------|---------|
| type | String | 消息类型 | "SYSTEM" |
| content | String | 创建结果描述 | "房间 [{房间名称}] 创建成功" |
| timestamp | Long | 时间戳（毫秒） | Unix时间戳 |

#### 注意事项

1. **权限要求**：仅管理员可用，非管理员用户会收到"权限不足"错误
2. **自动更新**：创建成功后会自动发送 ROOM_LIST 消息更新房间列表
3. **房间名称**：房间名称在系统中必须唯一
4. **私有房间**：私有房间需要设置密码，加入时需要提供正确的密码

#### 错误响应

| 错误信息 | 说明 | 解决方法 |
|---------|------|---------|
| 只有管理员才能创建房间 | 当前用户不是管理员 | 使用管理员账户登录 |
| 房间名称不能为空 | content参数为空 | 提供有效的房间名称 |
| 房间名称已存在 | 房间名称已被使用 | 使用不同的房间名称 |
| 创建房间失败 | 服务器内部错误 | 稍后重试或联系管理员 |

### 5. DELETE_ROOM - 删除房间（管理员）

#### 功能说明

删除指定的聊天房间。仅管理员可用。删除成功后会自动发送 ROOM_LIST 消息更新房间列表。房间内所有用户会被移出房间。

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 约束条件 | 默认值 |
|--------|------|------|------|---------|--------|
| type | String | 是 | 消息类型，固定值 | 固定为 "DELETE_ROOM" | - |
| roomId | String | 是 | 房间ID | 纯数字字符串，长度1-20 | - |

#### 参数详细说明

**type**
- 固定值："DELETE_ROOM"
- 说明：标识此消息为删除房间请求
- 必填：是

**roomId**
- 类型：String
- 说明：要删除的房间ID
- 必填：是
- 格式：纯数字字符串
- 长度：1-20个字符
- 约束：房间必须存在
- 获取方式：通过 ROOM_LIST 消息获取房间列表

#### 响应参数

| 参数名 | 类型 | 说明 | 可能的值 |
|--------|------|------|---------|
| type | String | 消息类型 | "SYSTEM" |
| content | String | 删除结果描述 | "房间已删除" |
| timestamp | Long | 时间戳（毫秒） | Unix时间戳 |

#### 系统广播

删除成功后：
1. 自动发送 ROOM_LIST 消息更新房间列表
2. 房间内所有用户会被移出房间
3. 被移出的用户会收到系统通知

#### 注意事项

1. **权限要求**：仅管理员可用
2. **自动更新**：删除成功后会自动发送 ROOM_LIST 消息
3. **用户移出**：房间内所有用户会被移出房间
4. **不可恢复**：删除的房间无法恢复

#### 错误响应

| 错误信息 | 说明 | 解决方法 |
|---------|------|---------|
| 只有管理员才能删除房间 | 当前用户不是管理员 | 使用管理员账户登录 |
| 请指定房间ID | 未提供roomId参数 | 提供要删除的房间ID |
| 房间不存在 | 指定的房间不存在或已被删除 | 检查房间ID是否正确 |

### 6. ROOM_LIST - 获取房间列表

#### 功能说明

获取所有活跃的房间列表。返回的房间列表包含房间的详细信息，如房间名称、描述、用户数等。

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 约束条件 | 默认值 |
|--------|------|------|------|---------|--------|
| type | String | 是 | 消息类型，固定值 | 固定为 "ROOM_LIST" | - |

#### 参数详细说明

**type**
- 固定值："ROOM_LIST"
- 说明：标识此消息为获取房间列表请求
- 必填：是

#### 响应参数

| 参数名 | 类型 | 说明 | 可能的值 |
|--------|------|------|---------|
| type | String | 消息类型 | "ROOM_LIST" |
| rooms | Array | 房间列表 | 房间对象数组 |
| timestamp | Long | 时间戳（毫秒） | Unix时间戳 |

#### rooms 数组元素

| 参数名 | 类型 | 说明 | 可能的值 |
|--------|------|------|---------|
| roomId | String | 房间ID | 纯数字字符串 |
| name | String | 房间名称 | 1-50个字符 |
| description | String | 房间描述 | 0-200个字符 |
| maxUsers | Integer | 最大用户数 | 1-1000 |
| isPrivate | Boolean | 是否为私有房间 | true或false |
| userCount | Integer | 当前用户数 | 0-maxUsers |

#### 响应详细说明

**rooms**
- 类型：Array
- 说明：房间列表数组
- 元素类型：对象
- 数组长度：0-N（取决于系统中活跃房间的数量）

**roomId**
- 类型：String
- 说明：房间的唯一标识符
- 格式：纯数字字符串
- 用途：用于加入房间、删除房间等操作

**name**
- 类型：String
- 说明：房间名称
- 长度：1-50个字符

**description**
- 类型：String
- 说明：房间描述
- 长度：0-200个字符
- 可以为空

**maxUsers**
- 类型：Integer
- 说明：房间最大用户数
- 范围：1-1000
- 约束：当userCount达到maxUsers时，新用户无法加入

**isPrivate**
- 类型：Boolean
- 说明：是否为私有房间
- 可能值：
  - true：私有房间，需要密码才能加入
  - false：公开房间，任何人都可以加入

**userCount**
- 类型：Integer
- 说明：房间当前用户数
- 范围：0-maxUsers
- 约束：userCount <= maxUsers

### 7. USER_LIST - 获取用户列表

#### 功能说明

获取用户列表。可以指定房间获取该房间的用户列表，或不指定房间获取所有在线用户。

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 约束条件 | 默认值 |
|--------|------|------|------|---------|--------|
| type | String | 是 | 消息类型，固定值 | 固定为 "USER_LIST" | - |
| roomId | String | 否 | 房间ID | 纯数字字符串，长度1-20 | null |

#### 参数详细说明

**type**
- 固定值："USER_LIST"
- 说明：标识此消息为获取用户列表请求
- 必填：是

**roomId**
- 类型：String
- 说明：房间ID
- 必填：否
- 格式：纯数字字符串
- 长度：1-20个字符
- 约束：房间必须存在
- 默认值：null
- 行为：
  - 提供roomId：返回指定房间的用户列表
  - 不提供roomId：返回所有在线用户列表

#### 响应参数

| 参数名 | 类型 | 说明 | 可能的值 |
|--------|------|------|---------|
| type | String | 消息类型 | "USER_LIST" |
| roomId | String | 房间ID | 与请求参数一致或null |
| users | Array | 用户列表 | 用户对象数组 |
| timestamp | Long | 时间戳（毫秒） | Unix时间戳 |

#### users 数组元素

| 参数名 | 类型 | 说明 | 可能的值 |
|--------|------|------|---------|
| userId | String | 用户ID | 纯数字字符串 |
| username | String | 用户名 | 1-50个字符 |
| isOnline | Boolean | 是否在线 | true或false |
| isAdmin | Boolean | 是否为管理员 | true或false |
| isMuted | Boolean | 是否被禁言 | true或false |

#### 响应详细说明

**roomId**
- 类型：String
- 说明：房间ID
- 可能值：
  - 与请求参数中的roomId一致（当提供roomId时）
  - null（当未提供roomId时）

**users**
- 类型：Array
- 说明：用户列表数组
- 元素类型：对象
- 数组长度：0-N

**userId**
- 类型：String
- 说明：用户的唯一标识符
- 格式：纯数字字符串
- 用途：用于踢出用户、禁言用户等操作

**username**
- 类型：String
- 说明：用户名
- 长度：1-50个字符

**isOnline**
- 类型：Boolean
- 说明：用户是否在线
- 可能值：
  - true：用户当前在线
  - false：用户当前离线

**isAdmin**
- 类型：Boolean
- 说明：用户是否为管理员
- 可能值：
  - true：用户是管理员
  - false：用户是普通用户

**isMuted**
- 类型：Boolean
- 说明：用户是否被禁言
- 可能值：
  - true：用户被禁言，无法发送聊天消息

## 错误处理

### 错误消息格式

| 参数名 | 类型 | 说明 | 可能的值 |
|--------|------|------|---------|

### 常见错误

| 错误信息 | 说明 | 解决方法 |
|---------|------|---------|
| 请提供Token | 连接时未提供Token | 在连接URL中添加 `token` 参数 |
| 认证失败 | Token无效或已过期 | 重新登录获取新Token |
| 请指定房间ID | 加入房间时未提供房间ID | 在请求中添加 `roomId` 字段 |

| 房间密码错误 | 私有房间密码错误 | 提供正确的房间密码 |
| 请先加入房间 | 在未加入房间时发送聊天消息 | 先发送 JOIN 消息加入房间 |
| 您已被禁言 | 用户被禁言，无法发送消息 | 等待禁言结束或联系管理员 |
| 权限不足 | 尝试执行需要管理员权限的操作 | 确保当前用户是管理员 |
| 消息包含敏感词 | 消息包含敏感词 | 移除敏感词后重新发送 |
| 撤回失败: 只能撤回自己发送的消息 | 尝试撤回他人的消息 | 只能撤回自己发送的消息 |
| 撤回失败: 消息发送时间已超过2分钟 | 消息发送时间超过2分钟 | 只能撤回2分钟内的消息 |
| 文本不能为空 | 敏感词检测时未提供文本 | 提供要检测或过滤的文本内容 |

## 敏感词检测

### 检测算法

系统使用三种敏感词检测算法：

1. **KMP算法**（单模式匹配）
   - 适用于单个敏感词的精确匹配
   - 时间复杂度：O(n+m)
   - n为文本长度，m为敏感词长度

2. **TRIE算法**（前缀树）
   - 适用于多个敏感词的快速匹配
   - 时间复杂度：O(n×m)
   - n为文本长度，m为敏感词平均长度

3. **AC自动机**（推荐）
   - 适用于大量敏感词的高效匹配
   - 时间复杂度：O(n)，n为文本长度
   - 系统默认使用此算法

### 检测流程

1. 用户发送聊天消息
2. 系统使用AC自动机检测敏感词
3. 如果检测到敏感词，拒绝消息并返回错误
4. 如果未检测到敏感词，保存并广播消息

### 敏感词管理

通过REST API管理敏感词：

- 获取所有敏感词：`GET /api/sensitive-words`
- 添加敏感词：`POST /api/sensitive-words`
- 删除敏感词：`DELETE /api/sensitive-words/{word}`
- 批量导入：`POST /api/sensitive-words/batch`
- 检测文本：`POST /api/sensitive-words/check`
- 过滤文本：`POST /api/sensitive-words/filter`

### 敏感词存储

- 敏感词存储在文本文件中，文件位置：`{工作目录}/sensitive_words.txt`
- 每行一个敏感词，使用UTF-8编码
- 系统每60秒自动从文件重新加载敏感词，修改文件后无需手动重载

## 注意事项

1. **连接认证**: 必须使用Token认证方式
2. **房间管理**: 用户必须先加入房间才能发送聊天消息
3. **权限控制**: 管理员功能（CREATE_ROOM, DELETE_ROOM, KICK, MUTE, UNMUTE, ADMIN）仅管理员可用
4. **消息撤回**: 只能撤回自己发送的消息，且消息发送时间不超过2分钟
5. **私有房间**: 加入私有房间需要提供正确的密码
6. **连接管理**: 连接断开后需要重新连接，会话状态不会保留
7. **消息格式**: 所有消息必须是有效的JSON格式
8. **时间戳**: 建议在发送消息时包含 `timestamp` 字段
9. **错误处理**: 始终监听 `ERROR` 类型的消息以处理错误情况
10. **心跳保活**: 建议实现心跳机制以保持连接活跃
11. **敏感词检测**: 所有聊天消息都会经过敏感词检测，包含敏感词的消息会被拒绝
12. **历史消息**: 加入房间后会收到最近10条历史消息
13. **ID格式**: 所有ID为纯数字字符串，使用UUID生成的数字ID
14. **WebSocket敏感词检测**: 支持通过WebSocket协议进行敏感词检测（CHECK_SENSITIVE和FILTER_SENSITIVE消息类型），功能与REST API一致
15. **敏感词自动重载**: 系统每60秒自动从文件重新加载敏感词，修改文件后无需手动重载
